# Planning Implementation

How the autonomous work loop operates with issues and PRs.

## Labels as State Machine

| Label | State | Action |
|-------|-------|--------|
| `needs-planning` | Has findings, needs checklist | Transform findings → Acceptance Criteria |
| `claude` | Has checklist, ready to execute | Execute next unchecked item |
| (none) | Waiting for PR merge | `next.yml` will re-add `claude` |

## Architecture (Mike Model)

```
Issue #63 (User Story)
├── ## Findings (original analysis)
├── ## Acceptance Criteria (checklist)
│   - [x] Item 1 ← PR #70 (merged)
│   - [x] Item 2 ← PR #71 (merged)
│   - [ ] Item 3 ← next to execute
│   - [ ] Item 4
└── closes when all checked
```

- **One issue** - No subtask issues, just checklist
- **Multiple PRs** - Each item = one PR
- **PRs reference issue** - "Fixes #63"

## Planning Mode

When issue has `needs-planning`:

1. Read findings in issue body
2. Check current codebase state
3. Transform findings → Acceptance Criteria checklist
4. Update issue body with `## Acceptance Criteria`
5. Swap labels: `needs-planning` → `claude`

If nothing to do: close issue.

## Execution Mode

When issue has `claude`:

1. Read Acceptance Criteria checklist
2. Find first unchecked item (`- [ ]`)
3. Execute that ONE item
4. Create branch + commits
5. Remove `claude` label (workflow will create PR)

## PR Format

The `claude.yml` workflow auto-creates PR with:

```markdown
Fixes #[issue-number]

---
Generated by [@synerbot](https://github.com/apps/synerbot)
```

## The Feedback Loop

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  Workflow creates Issue #63 + [needs-planning]           │
│                        │                                 │
│                        ▼                                 │
│  claude.yml → Planning Mode                              │
│    • Findings → Acceptance Criteria                      │
│    • Swap: [needs-planning] → [claude]                   │
│                        │                                 │
│                        ▼                                 │
│  claude.yml → Execution Mode                             │
│    • Execute first - [ ] item                            │
│    • Create branch + commits                             │
│    • Remove [claude]                                     │
│                        │                                 │
│                        ▼                                 │
│  claude.yml (post-step)                                  │
│    • Auto-create PR ("Fixes #63")                        │
│                        │                                 │
│                        ▼                                 │
│  👤 Human reviews & merges PR                            │
│                        │                                 │
│                        ▼                                 │
│  next.yml triggers                                       │
│    • Check if more - [ ] items                           │
│    • Yes → add [claude] ─────────────────────────────────►
│    • No → close issue                                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## Workflows

| Workflow | Trigger | Action |
|----------|---------|--------|
| `skill-review.yml` | Schedule | Creates issue + `needs-planning` |
| `daily-standup.yml` | Schedule | Creates issue + `needs-planning` |
| `claude.yml` | Label added | Planning or Execution + auto-PR |
| `next.yml` | PR merged | Re-adds `claude` or closes issue |

## Why This Design

1. **Simple** - One issue, one checklist, multiple PRs
2. **Traceable** - Clear issue → PRs relationship
3. **Agile** - Work through items one at a time
4. **Human-in-the-loop** - Only PR approval required
5. **No bureaucracy** - No subtask issues to manage
