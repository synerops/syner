# syner as operating system

mira este repo como las cosas que se pueden crear en syner:

- puedo crear "agents"
- puedo crear "apps"
- puedo crear "skills"
- puedo crear "packages"

`.syner` es el sistema operativo interno, puede estar en la raiz y en cada app y cada package.

## cada folder repite el proceso

- cada app, puede tener sus skills y sus agentes
- cada package, actualmente puede tener skills, a futuro podria tener agents

## todo funciona a partir de symlinks

los symlinks acomodan todo de tal forma que los coding agents lo puedan cargar como ellos lo necesitan
