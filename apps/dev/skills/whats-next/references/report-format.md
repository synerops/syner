# Report Format

Triage reports are written to `.syner/reports/triage-{date}.md`.

## Directory Structure

```
.syner/
  reports/
    triage-2024-01-15.md
    triage-2024-01-16.md
```

Create `.syner/reports/` if it doesn't exist.

## Report Template

```markdown
# Triage Report YYYY-MM-DD

## Your First Move

**[Specific action]** — [Why this matters and what it unblocks]

## Focus Area

**[Domain]** has [N] signals this week.

Related items:
- Backlog: [items]
- Tasks: [items]
- Issues: [items]

## Decisions Blocking Work

| Issue | Title | Days Waiting | Impact |
|-------|-------|--------------|--------|
| #X | Title | N | Blocks Y items |

## Priority Items

### Top Priority — Work on today
- [ ] [Item]
- [ ] [Item]

### This Week
- [ ] [Item]
- [ ] [Item]

### Later
- [ ] [Item]

## Cross-Reference Map

| Pattern | Sources | Priority |
|---------|---------|----------|
| Auth flow | 3 frictions, 1 backlog, 1 issue | Top priority |
| Build perf | 2 frictions, 1 task | This week |

## Sources Summary

| Source | Count | Notes |
|--------|-------|-------|
| Backlog items | X | From Y files |
| Grow observations | X | Last 7 days |
| Tasks | X | Y in progress |
| GitHub issues | X | Y need decision |

## Skipped Sources

- [Source]: [Reason] (e.g., "Observations: No .syner/ops/ directory")

---
Generated by /whats-next v0.3.0
```

## File Naming

- Format: `triage-YYYY-MM-DD.md`
- One report per day (overwrite if run multiple times)
- Use local date from `date +%Y-%m-%d`

## Console Output

After writing the report, output a brief summary to console:

```
Triage complete. Report: .syner/reports/triage-2024-01-15.md

First move: [action]
Focus area: [domain] ([N] signals)
Items: [X] top priority, [Y] this week, [Z] later
```

## Historical Reports

Keep reports for reference. Don't delete old reports automatically.

Use historical reports to:
- Track how many top-priority items clear per week
- Identify recurring patterns
- Measure friction reduction over time
